# 🎬 Viral Shorts Engine - 官方使用手册 (User Manual)

## 1. 简介
Viral Shorts Engine 是一款基于本地化的端到端 AI 短剧全自动生产工作流。它通过“网页劫持（Web Automation）”技术，零成本调用 Google Gemini、豆包（Doubao）、Nano Banana Pro 和 Veo 3.1 等前沿大模型，实现从“脑洞灵感”到“剧本”，再到“分镜视频”的工业化流水线。

## 2. 基础准备
请确保您已经完成了基础的环境配置与启动工作（安装 Node.js, FFmpeg, 以调试模式开启 Chrome 等）。
> 详细的环境部署和启动步骤，请务必优先参考：👉 [新手保姆级教程 (Tutorial)](./tutorial.md)

### 🚨 必须保持开启的网页
在启动项目终端之前，请务必在以**调试模式（9222端口）**开启的 Chrome 浏览器中，保持以下三个网页处于**登录且打开**的状态：
1. **Google Gemini** (`gemini.google.com`) 或 **豆包** (`doubao.com`)：负责剧本创作与提示词推演。
2. **Google Labs Flow** (`labs.google/fx/tools/flow/project/...`)：负责生图与生视频。
3. **Google AI Studio** (`aistudio.google.com/...`)：负责 TTS 语音合成。

## 3. 核心生产流程详解

进入 `http://localhost:3000` 后，你会看到四个核心的工作室，它们代表了严格的线性生产制片流程。

### 📍 阶段一：剧本室 (Phase 1)
这是创意爆发的地方。
- **配置大模型**：点击右上角设置(⚙️)，可自由切换 `Gemini` 或 `豆包 (Doubao)` 作为全局文本大脑。这决定了接下来所有文字工作由谁来完成。
- **创作灵感**：可以一键拉取热门网络段子，或者自己输入一个想法。
- **生成剧本**：AI 会自动输出带有极限反转的叙事剧本，并进行系统内打分评审。如果不合格，它会根据反馈自我迭代直到通过。
- **拆解分镜**：剧本定稿后，点击“拆解为分镜”，系统会自动提取出纯净的【角色表】和具体的【分镜台词表】。

### 📍 阶段二：定妆室 (Phase 2 - 极其关键！)
在这里，我们要为全剧生成“视觉锚点”（角色长什么样、场景长什么样）。**这步决定了你后续视频会不会变脸、场景会不会乱飞。**
1. **场景概念图**：点击生成场景提示词，系统会秉持“极简原则”为您生成一个空旷干净的背景图提示词（如：只有一张桌子和两把椅子的白墙房间，避免后续人物互动被挡住）。生图后，**请将这张满意的背景图保存下载**。
2. **角色定妆照**：为每一位出场的主角生成一张“正面、高清、纯净背景”的全身照，**同样请保存下载**。

### 📍 阶段三：Google Flow 素材绑定 (⭐️ 重点必看 ⭐️)
Viral Shorts Engine 是通过 `@` 符号系统来让 Flow 强制保持角色和场景连贯性的。

**您的手动操作规范：**
1. 切换到你的 **Google Labs Flow** 项目网页标签页。
2. 将你在【定妆室】保存的“场景图”和“角色图”，统统上传到 Flow 左侧的资产库（Assets）中。
3. **【严厉警告】必须在 Flow 中重命名资产（防伪标签对照表）！** 
   为了防崩溃防变脸，引擎通过插件提取图片时会自动复制一个“防伪名”。你在 Flow 中上传图片后，**必须重命名为对应提取时的防伪名**！
   这是系统底层硬编码的**铁律**，绝不可擅自修改：
   - **全局场景图 (locationImage)**：插件提取时强制复制为 **`场景`**。在 Flow 里必须叫 `场景`。
   - **角色定妆图 (characterImage)**：插件提取时会精确复制该角色的真实名字（例如 **`小雪`**、**`布偶猫`**）。
   - **画板首帧图 (sceneStartImage)**：提取时防伪名格式为 `项目名_S镜头号_StartImg`（例：`Proj_S0_StartImg`）。
   - **画板尾帧图 (sceneImage)**：提取时防伪名格式为 `项目名_S镜头号_Img`（例：`Proj_S0_Img`）。
4. **底层原理**：后续生成视频时，引擎如果在提示词中检测到 `{@小雪}`，它会在网页上飞速打出 `@小雪` 并按回车选中素材！**如果你在 Flow 里没改名，代码就会找不到素材，导致角色彻底变脸脱相！规则就是规则，请严格遵守。**

### 📍 阶段四：画板区 (Phase 3)
1. 此时系统会自动根据你的台词，一键打磨生成每个分镜的首尾帧静态图提示词（`startImagePrompt`, `imagePrompt`）以及视频动态描述（`videoPrompt`）。
2. 在这里，你可以点击每一镜的【生成视频】按钮。
3. 引擎会在后台接管 Flow 网页：
   - 自动填入提示词
   - 自动敲击 `@` 引用你命名好的场景和角色
   - 自动开启 Veo 3.1 渲染出带有极其稳定微动作的视频片段。
4. 系统采用了“首尾帧继承架构”（即第2镜的首帧自动使用第1镜的尾帧），你无需进行任何干预，坐等渲染即可。

### 📍 阶段五：渲染室 (Phase 4)
1. 当所有的视频片段（MP4）全部拿到手后，进入渲染室。
2. 系统会自动生成所有角色的 TTS 语音。
3. 拖动时间轴上的音频轨道，将配音和人物动作对齐。
4. 点击【合成导出 4K】，引擎会使用本地的 FFmpeg 把视频、配音、字幕完美压制成一部可直接发布的竖屏 4K 爆款短剧！

## 4. 常见问题排查 (FAQ)

**Q1：为什么视频里的主角不长我设定的样子？**
**A1**：绝对是因为你在 Google Flow 侧边栏没有把角色图片的名称改成对应的中文名。系统敲击 `@小雪` 时没搜到你的图。请检查拼写是否与剧本里的角色名**一字不差**。

**Q2：为什么视频的背景每次都在变？**
**A2**：同上。必须确保你在 Flow 里有一张刚好命名为 `场景` 的空镜底图，并且在画板区的提示词里有 `{@场景}` 这个标记。

**Q3：为什么代码运行时在 Flow 网页里卡住了，并且跳出“异常活动”？**
**A3**：由于自动化脚本跑得太快，你被 Google 判定为机器人并风控限制了 IP。
**解法**：请清空 Chrome 缓存（Shift+Command+Delete），刷新页面，并挂上纯净的 WARP 全局代理节点重试。详情参考：[WARP 隧道防封禁指南](./warp-proxy.md)。

**Q4：两个人对话的场景，为什么画面里只有一个人在对着空气说话？**
**A4**：检查【画板区】生成的中文提示词中，是否把两个角色都 `@` 进去了。如果 AI 偷懒漏写了，请手动修改提示词，把没出现的人补上去，例如加上：`{@角色A}与{@角色B}面对面站着`。只有同时被 `@` 的人物才会出现在同一个画面里（Two-Shot）。
