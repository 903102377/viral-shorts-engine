# 提示词与架构约定 (Prompt Conventions)

这篇文档记录了 Viral Shorts Engine 中**极其重要**的提示词生成约束和系统规则。
以后**任何 AI 助手或开发者**在重构、修改提示词模板（`actionTemplates.ts`, `characterTemplates.ts` 等）时，**必须绝对遵守以下天规**，否则会导致系统级崩溃或严重的生图生视频 Bug。

## 1. 场景锚定与 Token 语法 (极其关键)
为了保证同一场景下的背景画面一致性，系统使用硬编码的 Token 进行图片占位。
- **强制使用固定的 `{@场景}`**：
  在 `imagePrompt` 和 `startImagePrompt` 的开头或环境描写处，**只能使用 `{@场景}` 这四个字**。
  - ❌ 错误示范：`{@银行大厅}`、`{@我的卧室}`
  - ✅ 正确示范：`在{@场景}中...`
  *原因*：底层前端逻辑及拦截器**仅能严格识别并替换** `{@场景}` 这个固定字符串，用来映射全局场景参考图。一旦 AI 自作主张改名，所有背景图注入将直接报废！

## 2. 媒体资产(Assets)命名铁律（不可更改！）
Chrome 扩展（`content.js`）及后端自动生成文件时，必须严格遵守以下标识符，**任何开发者不得乱改此规则**：
- `locationImage` (场景概念图)：目标名称强制锁定为 **`场景`**。
- `characterImage` (角色定妆照)：目标名称强制锁定为 **角色原始全名**（如 `小雪`）。
- `sceneStartImage` (分镜首帧图)：标识符必须为 `[项目名]_S[序号]_StartImg`。
- `sceneImage` (分镜尾帧图)：标识符必须为 `[项目名]_S[序号]_Img`。
- `sceneVideo` (分镜视频)：标识符必须为 `[项目名]_S[序号]_Vid`。

## 2. 角色锚定语法与同框逻辑
- **角色 Token `{@角色名}`**：
  同理，在 `imagePrompt` 和 `startImagePrompt` 中，凡是出现角色的地方，必须使用 `{@名称}` 包裹（例如 `{@布偶猫}`，`{@柯基柜员}`）。
  *原因*：这是为了让 Nano 等模型能精准识别人物并使用面部/角色特征的参考图，确保长视频中人物的脸不崩。
- **多角色同框（Two-Shot 强制逻辑）**：
  如果当前的剧本动作暗示了双方互动（例如对话、递东西），AI 必须在生成首尾帧画面描述时，**把参与互动的所有角色都写进画面里**（例如隔着玻璃对峙的猫和柜员），构成双人同框镜头。绝对禁止“让对手隐身，导致当前角色对着空气说话”的单人镜头幻觉。

## 3. 视频生成逻辑中的避坑指南 (AI 视频的物理局限性)
- **videoPrompt 中绝对禁用 `{@}` 语法**：
  `videoPrompt` 是喂给 Veo 3.1 等视频生成大模型的纯中文动态描述，里面**不支持且不能包含**任何带有 `{}` 或 `@` 的系统 Token。如果在 `videoPrompt` 中出现 `{@场景}`，视频生成的文字解析会出错或把这些奇怪的符号直接糊到画面上。直接使用自然语言描写环境即可。
- **强制微动作与平稳运镜 (极度重要)**：
  千万不要在 `videoPrompt` 里写“猛然窜起”、“重重拍击”、“镜头急推”、“物理震颤”这种激烈的动作或运镜！AI 视频模型（如 Veo 3.1/Sora）处理剧烈动作必崩必变形。所有动作必须转化为**克制、缓慢的微动作**（例如：“缓慢而坚定地将黑卡推向防弹玻璃”）。镜头必须规定为“固定镜头”或“极其缓慢的推移”。
- **禁止生硬的格式化前缀**：
  不要用“连贯动作开始：”、“运镜配合：”这种说明书式的生硬前缀。必须写成一段如电影剧本般流畅连贯的纯中文自然段落。

## 4. 首尾帧架构 (Start/End Frame)
我们的工作流利用了高级视频生成的首尾双帧控制机制来保证连贯性：
- **第 1 镜特权**：必须同时生成 `startImagePrompt`（首帧图）和 `imagePrompt`（尾帧图）。
- **第 2+ 镜继承原则**：系统前端会自动拿**上一镜的尾帧图作为本镜的首帧**！因此 AI 在生成第 2 镜及以后镜头的提示词时：
  1. 绝对不需要生成 `startImagePrompt`。
  2. AI 必须读取【上一镜的尾帧图描述】作为起始上下文，生成的 `imagePrompt`（本镜尾帧）和 `videoPrompt`（动态过程）**必须基于并延续**上一镜结束时的状态（例如：上一镜角色是举起手，这一镜的起点就是举起手，不能莫名其妙归位）。

## 5. 画风与质感要求 (Raw Realism)
本项目已从早期的“Pop Mart 盲盒/3D可爱风”全面升级为 **“极高保真的原生写实主义 (Raw Realism)”**：
- 在生成角色定妆照、全局场景概念图以及最终的分镜画面提示词时，必须明确强调：“不完美之美 (imperfect beauty)”、“不磨皮、不美颜、无滤镜”、“真实可见的皮肤微小瑕疵（如毛孔）”、“真实的居家自然光影不均”等。
- 坚决杜绝 AI 习惯性的“精美海报感”、“商业打光”或任何像 AI 画作的痕迹。

## 6. 全中文输出 (Chinese-Only)
- 结合底层生图接口对本土化细节的卓越理解能力，目前引擎的所有生图、生视频 Prompt **全部强制要求输出纯中文**（除极个别底层保留的英文参数）。这样既能提高 AI 在处理中国文化背景时的写实感，也方便用户直接在 UI 界面上审查和微调。
